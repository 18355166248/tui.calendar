# 템플릿

## 설명

템플릿은 커스텀 렌더링을 지원하는 기능이다. 캘린더 인스턴스 생성 시 템플릿 옵션으로 커스텀 렌더링을 할 수 있으며, `setOptions`로 템플릿 옵션을 변경할 수 있다.

```jsx
const cal = new Calendar('#container', {
  template: {
    milestone(event) {
      return <span style={{ color: 'red' }}>{event.title}</span>;
    },
  },
});

cal.setOptions({
  template: {
    milestone(event) {
      return <span style={{ color: 'blue' }}>{event.title}</span>;
    },
  },
});
```

### 월간뷰의 더보기 팝업 관련

![more-events-popup](https://user-images.githubusercontent.com/28647773/169254478-f625bbe0-6b4f-4783-a8a5-560b1b7a26d4.png)

#### monthMoreTitleDate

```ts
interface TemplateMoreTitleDate {
  ymd: string;
  date: number;
  day: number;
}
```

더 보기 팝업의 날짜를 커스터마이징할 수 있다. 파라미터는 `TemplateMoreTitleDate` 객체로, `ymd` 프로퍼티는 해당 날짜의 `YYYY-MM-DD` 문자열 형식 데이터다. `date` 프로퍼티는 해당 날짜의 일, `day`는 해당 날짜의 요일이다.

#### monthMoreClose

더 보기 팝업의 닫기 버튼을 커스터마이징할 수 있다. 기본 값이 빈 문자열을 반환하기 때문에 기본적으로는 닫기 버튼이 표시되지 않는다.

### 월간뷰의 header, footer 관련

![grid-header-footer](https://user-images.githubusercontent.com/28647773/169254710-c190ef82-5525-4f55-a4f7-3aebef20f306.png)

```ts
interface TemplateMonthGrid {
  date: string;
  day: number;
  hiddenEventCount: number;
  isOtherMonth: boolean;
  isToday: boolean;
  month: number;
  ymd: string;
}
```

#### monthGridHeader

월간뷰 각 셀의 헤더의 날짜를 커스터마이징할 수 있다. `TemplateMonthGrid` 객체를 파라미터로 받는다.
`date`는 해당 날짜의 일, `day`는 해당 요일, `month`는 해당 월, `hiddenEventCount`는 표시되지 않은 이벤트의 갯수, `isOtherMonth`는 현재 월간뷰의 달과 다른 달의 날짜인지 여부, `isToday`는 오늘 날짜인지 여부, `ymd`는 해당 날짜의 `YYYY-MM-DD` 문자열 형식 데이터다.

#### monthGridHeaderExceed

월간뷰 각 셀의 헤더의 초과되는 이벤트의 갯수를 표시하는 컴포넌트를 커스터마이징할 수 있다. 초과된 이벤트의 갯수를 파라미터로 받는다.

#### monthGridFooter

월간뷰 각 셀의 푸터의 날짜를 커스터마이징할 수 있다. `TemplateMonthGrid` 객체를 파라미터로 받는다.
`date`는 해당 날짜의 일, `day`는 해당 요일, `month`는 해당 월, `hiddenEventCount`는 표시되지 않은 이벤트의 갯수, `isOtherMonth`는 현재 월간뷰의 달과 다른 달의 날짜인지 여부, `isToday`는 오늘 날짜인지 여부, `ymd`는 해당 날짜의 `YYYY-MM-DD` 문자열 형식 데이터다.
기본 값이 빈 문자열을 반환하기 때문에 따로 설정하지 않는다면 표시되지 않는다.

#### monthGridFooterExceed

월간뷰 각 셀의 푸터의 초과되는 이벤트의 갯수를 표시하는 컴포넌트를 커스터마이징할 수 있다. 초과된 이벤트의 갯수를 파라미터로 받는다.
기본 값이 빈 문자열을 반환하기 때문에 따로 설정하지 않는다면 표시되지 않는다.

주간뷰의 각 요일을 커스터마이징할 수 있다. `date`는 해당 요일의 일, `day`는 해당 요일, `dayName`은 해당 요일의 기본적인 영문 축약형 문자열, `isToday`는 해당 요일이 오늘인지 여부, `renderDate`는 주간뷰 렌더링의 기준 날짜, `dateInstance`는 해당 요일의 `Date` 객체다.
